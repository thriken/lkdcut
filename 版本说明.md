# 玻璃数据库删除工具 - 优化版本说明

## 版本概览

项目提供了三个不同级别的优化版本，满足不同使用场景的需求。

### 1. dbtool-simplified.py - 简化版
**特点：**
- ✅ 纯本地数据库操作
- ✅ 基本缓存机制
- ✅ 简洁的线程管理
- ✅ 防重复连接
- ✅ 自动大写转换

**适用场景：**
- 只需要基本功能的用户
- 对性能要求不高的简单场景
- 资源受限的环境

### 2. dbtool-optimized.py - 优化版
**特点：**
- ✅ 网络环境自动检测
- ✅ 高级缓存系统
- ✅ 连接池管理
- ✅ 智能搜索优化
- ✅ 批量删除性能

**适用场景：**
- 需要在不同网络环境中使用的用户
- 对性能有一定要求的场景
- 中等复杂度的数据库操作

### 3. dbtool-advanced.py - 高级优化版（推荐）
**特点：**
- ✅ 高级连接池管理（单例模式）
- ✅ 智能缓存系统（TTL + LRU）
- ✅ 实时性能监控
- ✅ 错误重试机制
- ✅ 可配置的性能参数
- ✅ 图形化配置界面
- ✅ 实时状态显示

**适用场景：**
- 高性能要求的专业场景
- 频繁进行数据库操作的用户
- 需要详细性能监控的场景

## 数据库结构说明

### 主要表结构

**file_info 表：**
- id: INTEGER (主键)
- directory: TEXT (文件目录)
- file_name: TEXT (文件名)
- file_md5: TEXT (文件MD5)

**glass_data 表：**
- id: INTEGER (主键)
- file_id: INTEGER (关联file_info.id)
- 各种玻璃切割相关字段（23个字段）

## 使用建议

### 对于新用户
1. **初级用户** → 使用 `dbtool-simplified.py`
2. **中级用户** → 使用 `dbtool-optimized.py`  
3. **高级用户** → 使用 `dbtool-advanced.py`

### 性能调优建议

1. **缓存配置**
   - 搜索缓存：默认200项，10分钟过期
   - 计数缓存：默认500项，5分钟过期
   - 可根据实际使用情况调整缓存大小

2. **连接池配置**
   - 自动清理超时连接（10分钟无操作）
   - 支持错误重试（最多2次）

3. **搜索优化**
   - 实时搜索防抖（可配置延迟）
   - 智能缓存只针对中小规模结果

## 技术架构

### 核心组件

1. **连接池管理器** (ConnectionPool)
   - 单例模式确保全局唯一
   - 自动连接生命周期管理
   - 超时连接自动清理

2. **高级缓存系统** (AdvancedCache)
   - TTL（生存时间）机制
   - LRU（最近最少使用）淘汰策略
   - 智能缓存大小控制

3. **性能监控装饰器** (performance_monitor)
   - 函数执行时间跟踪
   - 性能数据统计分析
   - 错误处理与重试机制

4. **线程管理**
   - 搜索线程 (SearchThread)
   - 删除线程 (DeleteThread)
   - 安全的线程间通信

## 安装与运行

### 依赖安装
```bash
pip install PyQt5
```

### 运行方式
```bash
# 简化版
python dbtool-simplified.py

# 优化版  
python dbtool-optimized.py

# 高级版（推荐）
python dbtool-advanced.py
```

### 环境要求
- Python 3.6+
- PyQt5 库
- SQLite3 数据库文件 (glass_data.db)

## 故障排除

### 常见问题

1. **数据库连接失败**
   - 检查 glass_data.db 文件是否存在
   - 确认文件权限设置正确
   - 检查文件是否被其他程序占用

2. **搜索性能问题**
   - 启用缓存功能
   - 调整搜索延迟参数
   - 定期清空缓存

3. **内存使用过高**
   - 减少缓存大小设置
   - 定期重启程序
   - 使用性能监控功能分析
